<objective>
Configure multi-model routing: DeepSeek V4 Pro for root-level complex reasoning, DeepSeek V4 Flash for sub-calls.

Purpose: Pro is 3x more expensive but better at complex tasks. Flash is fast and cheap for sub-queries (llm_query, rlm_query at depth > 0). Automatic depth-based routing saves cost without sacrificing quality.
Output: Updated config.py with dual-model settings, updated rlm_client.py with other_backends configuration.
</objective>

    if model and model in self.clients:
        return self.clients[model]        # Explicit model override
    if depth == 1 and self.other_backend_client:
