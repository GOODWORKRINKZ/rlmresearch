  <name>Task 1: Create tools.py with workspace-sandboxed tools</name>
  <files>src/rlm_assistant/tools.py</files>
  <action>
Create `src/rlm_assistant/tools.py` with a `build_custom_tools(workspace_dir: str) -> dict` factory function.

The factory returns a dict in RLM's custom_tools format with 4 tools:

1. **read_file(path: str) -> str** — Read file contents.
   - Resolve path relative to workspace_dir using Path.resolve()
   - Reject paths outside workspace (check str(resolved).startswith(str(workspace)))
   - Return file contents as string, handle encoding errors with errors="replace"
   - On error: return "ERROR: ..." message (never raise)

2. **write_file(path: str, content: str) -> str** — Write content to file.
   - Same path sandboxing as read_file
   - Create parent directories automatically (mkdir parents=True, exist_ok=True)
   - Return f"Wrote {len(content)} bytes to {path}"
   - On error: return "ERROR: ..." message

3. **run_command(cmd: str) -> str** — Run shell command.
   - Use subprocess.run(cmd, shell=True, capture_output=True, text=True, timeout=30, cwd=workspace_dir)
   - Combine stdout and stderr
   - Return "(no output)" if both empty
   - Catch subprocess.TimeoutExpired: return "ERROR: Command timed out (30s limit)"
   - Catch other exceptions: return "ERROR: {e}"

4. **search_code(pattern: str, path: str = ".") -> str** — Regex search in files.
   - Resolve and sandbox the target directory
   - Walk file tree with Path.rglob("*")
   - Filter by common code extensions: .py, .ts, .js, .rs, .md, .json, .yaml, .yml, .toml, .txt
   - Use re.search(pattern, line, re.IGNORECASE) for matching
   - Return matches as "{relative_path}:{line_num}: {line.strip()}"
   - Limit to first 100 matches to avoid overwhelming output
   - On error: return "ERROR: {e}"

Each tool entry in the returned dict uses the dict format:
```python
{
    "read_file": {"tool": _read_file, "description": "Read file contents. Usage: read_file('path/to/file')"},
    "write_file": {"tool": _write_file, "description": "Write content to file. Usage: write_file('path/to/file', 'content')"},
    "run_command": {"tool": _run_command, "description": "Run shell command (30s timeout). Usage: run_command('ls -la')"},
    "search_code": {"tool": _search_code, "description": "Search codebase for pattern. Usage: search_code('def main', 'src/')"},
}
```

Import only stdlib: `pathlib.Path`, `subprocess`, `re`. No external dependencies.

Add module docstring explaining the tools module purpose.
