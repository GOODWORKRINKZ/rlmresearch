  <action>
    Create src/rlm_assistant/server.py with a FastAPI application.

    **Pydantic models** (define at top of file):

    ChatMessage: role (str), content (str)
    ChatCompletionRequest: model (str), messages (list[ChatMessage]), stream (bool = False), temperature (float = 0.7), max_tokens (int | None = None)
    ChatCompletionChoice: index (int), message (ChatMessage), finish_reason (str)
    ChatCompletionResponse: id (str), object (str = "chat.completion"), created (int), model (str), choices (list[ChatCompletionChoice])

    **Endpoints:**

    1. POST /v1/chat/completions:
       - Accept ChatCompletionRequest
       - Extract the last user message from request.messages (filter by role == "user")
       - Call chat(last_user_message.content) from rlm_client module
       - Build ChatCompletionResponse with:
         id: f"chatcmpl-{uuid.uuid4().hex[:12]}"
         created: int(time.time())
         model: request.model (echo back)
         choices: [ChatCompletionChoice(index=0, message=ChatMessage(role="assistant", content=response), finish_reason="stop")]
       - Return the response dict

    2. GET /v1/models:
       - Return {"data": [{"id": "rlm-dev-assistant", "object": "model", "created": 0, "owned_by": "rlm"}]}
       - This lets VS Code Copilot discover the model

    3. GET /health:
       - Return {"status": "ok", "backend": "deepseek", "model": settings.deepseek_model}

    **CORS middleware:**
    - Add CORSMiddleware with allow_origins=["*"], allow_methods=["*"], allow_headers=["*"]
    - Required for VS Code to connect from localhost

    **main() function:**
    - Import uvicorn
    - Call uvicorn.run(app, host=settings.host, port=settings.port)
    - This is the CLI entrypoint referenced in pyproject.toml [project.scripts]

    **Startup event** (@app.on_event("startup")):
    - Call get_rlm() to eagerly initialize the RLM singleton
    - Log "RLM server starting with DeepSeek backend" via logging

    Use logging module (logger = logging.getLogger(__name__)) for request/error logging.

    NOTE: Streaming is Phase 4. Phase 1 returns full responses only (stream=False in request is fine, just return the complete response).
